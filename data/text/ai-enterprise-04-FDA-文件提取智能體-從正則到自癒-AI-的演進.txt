===== PAGE 14 =====
代碼解說：lab_01_regex.py（精準Regex）
Stage 1 的四個Pattern 假設PDF 文字結構永遠規則
• K_NUMBER_PATTERN 用[Kk]\d{6} 防止誤匹配，但K 和數字之間不能有空格
• precise_patterns 對應4 種FDA 標準格式（Predicate K number、510(k)...）
• 任何一個Pattern 都假設PDF 裡有明確的關鍵字且位置固定——格式一亂就全部失效
• 失敗時：靜默回傳([], [])，CSV 顯示N/A

===== PAGE 20 =====
LangGraph Agent 是整個FDA 
提取系統的核心大腦
為什麼while loop 不夠，State 物件才是關鍵
• while loop：每次重試，上次報錯消失
• LangGraph State：error_trace 共享
• 條件路由：Critic 通過→ END / 失敗
→ 攜帶報錯回Coder
• 自癒迴圈（方法論）× LangGraph（技術手段）
迭代進化：每次失敗，系統變得更聰明一點

===== PAGE 21 =====
代碼解說：Coder Node 長什麼樣？
error_message x Prompt →自癒出發點
• 前4 次Local LM Studio → 連線失敗自動切GPT-4o
• Fallback → 第5 次起GPT-4o Expert Mode
•  Prompt 設計, instruction 把error_message + feedback 塞入（
error_context 變數），讓AI 讀到診斷報告
•  理解這個Prompt 是AI 的診療單——它讀了什麼錯誤，才能改什麼策略

===== PAGE 22 =====
代碼解說：Critic Node：誰說提取成功了？
Critic 是確定性規則引擎，不是AI，這是整個架構最重要的設計決策
• 核心原則：驗證絕對不能交給AI——AI 可能說「成功了」但結果是空的
或格式錯誤
• 三層驗證：①非空檢查→ ②K-Number 格式正則→ ③可選白名單比對
• Critic 是系統的「誠實錨點」：不花任何Token，卻是整個迴圈能停下來
的原因
• Critic 回傳失敗時，error_message 被填入State，下一輪Coder 繼續改
寫

===== PAGE 26 =====
我們誠實面對的限制
自癒不是萬能，設定停損點交還給人類
• 5 次上限機制：如果AI 在兩個錯誤之間反覆橫跳，
必須強制中斷
• 人工審核：承認AI 有搞不定的極端案例，誠實交還
給工程師
• 不要讓系統假裝成功：這比拋出Exception 更加危險
