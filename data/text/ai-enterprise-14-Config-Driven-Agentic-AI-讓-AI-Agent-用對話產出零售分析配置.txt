===== PAGE 12 =====
過去怎麼做？為何需要 Agent？
每個客戶的業務邏輯不同，但程式碼無法接受自然語言作為輸入，因此需要一個 轉換步驟
業務邏輯（自然語言）
模糊、主觀的分析需求，例如「比較我們跟主要競
品的銷售狀況」
人工轉換
將業務邏輯解析為 分析單位 與 邊界，生成結構化的
設定檔
可被腳本執行的設定檔
程式碼可直接讀取 YAML / JSON 並執行分析
前提是業務想分析的單位和邊界，需要能夠透過 資料庫欄位 篩選出來。

===== PAGE 14 =====
Agent 作為調度者
我們把分析流程中會需要使用到的腳本先建好，讓 Agent 來銜接每個步驟
01
使用者需求
定義目標與範圍
02
工作流程與計畫
根據 template 產出規劃檔
workflow.yaml
03
執行計劃
探索、實際查詢、資料轉換、產出交付物
（包含洞察）
早期的設計裡，workflow.yaml 像一份給 Agent 看的流程說明書 ← SKILL.md。
它可以寫步驟、依賴、工具、輸入輸出，也可以包含自然語言。
Agent 本身就是調度者，負責理解現在做到哪一步、下一步要跑什麼、要怎麼填參數。
Reference: https://github.com/shopify/roast

workflow.yaml 的優點與代價

適合探索新流程

像 pseudo-code
⚠️ 代價
執行速度慢

token 成本高

上下文腐敗
穩定性差（信任問題）

===== PAGE 21 =====
分析合約 = 可被程式碼讀取的業務邏輯
業務邏輯 = 分析單位 Entity + 邊界 Filter
project.yaml
├── 客戶與專案
├── 時間範圍
├── analysis_type：P0 / P1 / P3
├── Entities：品牌 / 商品線 / 競品 / 市
場基準
├── Filters：item_id prefix、商品名
稱 pattern、排除條件
├── Market baseline：市佔率分母
└── visualization /output settings
分析單位 Entities 範例
鮮乳坊鮮乳
- entity_id: bettermilk_fresh_milk
  name: 鮮乳坊鮮乳
  display_name: "鮮乳坊"
  filter_patterns:
    carrefour_filter: |
      item.item_name LIKE '%鮮乳坊%'
      AND item.item_id LIKE '15010%'
鮮乳市場 - 小容量（<900ml）
  - entity_id: "fresh_milk_small_vol"
    name: "鮮乳 小容量 (<900ml)"
    name_en: "Fresh Milk - Small Volume (<900ml)"
    display_name: "鮮乳 小容量"
    status: "active"
    description: "鮮乳市場 200ml、228ml、229ml、
290ml、440ml 等小包裝品項"
    skip_market_share: true
    filter_patterns:
      carrefour_filter: |
        item.item_id LIKE '15010%'
        AND (
          item.item_name LIKE '%200ml%'
          OR item.item_name LIKE '%200ML%'
          OR item.item_name LIKE '%228ml%'
          OR item.item_name LIKE '%229ml%'
          OR item.item_name LIKE '%230ml%'
          OR item.item_name LIKE '%290ml%'
          OR item.item_name LIKE '%440ml%'
        )

能夠被篩選出來

設計分析單位

市場邊界
5
視覺化分組
決定哪些分析單位要呈現在同一張
圖表中

其他配置

現行能力：三種分析執行路徑
不是所有需求都一次產品化：用三種路徑處理不同成熟度
穩定性高
L1: 高頻、穩定、可模板化
Pipeline Script
腳本執行分析
穩定性中
L2: 中頻、半結構化
Analysis Skills
Agent 看使用說明書執行分析
穩定性低
L3: 低頻、高客製
Custom SQL
Agent 根據需求自由發揮
從左到右的需求頻率和穩定性：高 → 低
