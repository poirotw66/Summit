使⽤ Redis 快取
CACHE · 降低 DB Loading
將權限、Menu、Building 等經常查詢
但不常變更的資料放入 Redis,降低 DB 
負載;搭配過期時間維持資料彈性,確保
橫向擴展時的⼀致性。 
2
服務託管於 GKE
INFRA · ⾃動擴展
所有微服務皆託管於 GKE,當 CPU 或記
憶體使⽤率達⾨檻時⾃動擴增 Pod 數
量,動態調度資源,應對臨時⾼流量。 
3
JWT + HS256 驗證
AUTH · 無狀態驗證
使⽤者登入後, AuthServer 以 HS256 
產⽣ JWT;各服務皆可透過 API 向 
AuthServer 驗證權限,較傳統 Session 
更靈活,契合微服務架構。 
4
微服務 + 無狀態
ARCHITECTURE · 低耦合
開發之初即進⾏服務與資料拆分;無狀
態設計搭配 Redis, 多實例彈性擴展，
資料無縫同步
5
Secret Manager
SECURITY · IAM 授權
將 JWT 密鑰、DB 資訊、內部驗證等
機敏資料集中於 Secret Manager;透過 
Google IAM 控管讀取權限,⼤幅提升系
統安全性。 
6
建構於 GCP 平台
CLOUD · Google Cloud Platform
系統整體建構於 GCP 之上,整合 GKE、
Secret Manager、IAM、Cloud 
Memorystore 等原⽣服務,享有企業級
雲端基礎建設的穩定與安全。
