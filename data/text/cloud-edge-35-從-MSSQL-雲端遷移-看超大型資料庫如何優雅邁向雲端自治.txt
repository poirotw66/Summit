Reserve 
財務槓桿
Performance Tuning
效能槓桿
RI / Reserved Capacity / Hybrid Benefit
在既有規格上打折。用量不變，規格鎖定1–3 年
索引、查詢/執行計畫、統計資訊、減少邏輯讀取(I/O)
直接降低CPU / I/O / 記憶體需求，從底層降低SKU 成本
從源頭就可能被低估的成本槓桿– 效能調校
RI 是折扣(乘法)，效能調校是降用量(減法)。先將多消耗的用量消除，折扣才真正划算。

FORCE_LAST_GOOD_PLAN
偵測「計畫退化」，自動強制返回上一個
已知良好的執行計畫
CREATE INDEX
依工作負載自動建立索引，只在低負載時
建立並驗證效益
DROP INDEX
移除90 天未使用與重複索引；不刪唯一、
主鍵索引
Automatic Tuning 功能、限制與風險
預設開啟
預設關閉
預設關閉
風險
• 自動建索引增加寫入/儲存負擔,可能與既定索引策略衝突
• DROP INDEX 可能誤刪罕用索引(如月底報表)
• FORCE_LAST_GOOD_PLAN 對特定參數值未必最佳
限制
• Managed Instance 只支援FORCE_LAST_GOOD_PLAN
• 不會改動t-sql / schema / non-SARGable — 核心調教仍仰賴DBA

===== PAGE 31 =====
Key Takeaways
- 雲端成本最大的浪費來自於過度配置（Overprovisioning）
- 遷移策略與路徑，往往比搬去哪裡重要的多
- 資料庫通常是最昂貴且最容易被過度配置的雲端資源之一
- 持續效能優化Tuning 是FinOps 最重要的底層邏輯
- RI 只能讓你『便宜地浪費』;唯有Tuning 讓你『不再浪費』
