三個比例相乘 ×100：把「進度、速度、預測」壓成一個可比較的指數
風險指數 =（已支出金額比例 ÷ 預算時間期程比例）×（預測未來全期費用比例）× 100

===== PAGE 14 =====
公式裡的三個比例，分別是什麼？
■
① 已支出金額比例 ＝ 目前累計支出 ÷ 總預算
●
車子已經跑掉的油 — 走得多遠了
■
② 預算時間期程比例 ＝ 已過時間 ÷ 預算總期程
●
時間的進度條 — 這一年過了幾分之幾（年初到今天）
■
③ 預測未來全期費用比例 ＝ 預測全期總花費 ÷ 總預算
●
照現在的開法，到年底會花到預算的幾 %

===== PAGE 15 =====
風險指數 = 100 代表什麼？
風險指數 = 100：一切「剛剛好」的基準線
已支出比例 = 時間比例
（÷ 後 = 1，呈線性）
×
預測比例 = 100%
（剛好打平預算）
風險指數 = 1 × 1 × 100 = 100
支出與時間成線性、預測剛好用完 → 完全 on-track

===== PAGE 28 =====
借用股市的「均線」語言，解讀雲端花費
■
MA30 ≈ 月線（短期）：最近一個月的花費水位，反應最快、最敏感
■
MA90 ≈ 季線（中期）：一季的花費趨勢，濾掉短期雜訊
■
MA360 ≈ 年線（長期）：整年的花費基準 — 正好對應你的「全年預算期程」
■
黃金交叉（短線向上穿越長線）：花費開始加速，超支風險升高的早期警訊
■
死亡交叉（短線向下穿越長線）：花費降溫，最佳化開始見效
■
多頭排列 MA30>MA90>MA360 ＝ 持續升溫；均線糾結交織 ＝ 趨勢轉折，要留意

===== PAGE 30 =====
三條均線的 SQL：一次算出 MA30 / MA90 / MA360
-- 用同一份每日花費，
-- 開三個不同長度的視窗
WITH daily AS (
  SELECT charge_date,
         SUM(effective_cost) AS cost
  FROM billing
  GROUP BY charge_date
)
SELECT charge_date, cost,
  AVG(cost)         OVER w30  AS 
ma30,
  AVG(cost)         OVER w90  AS 
ma90,
FROM daily
WINDOW
  w30  AS (ORDER BY charge_date
     ROWS BETWEEN 29  PRECEDING AND 
CURRENT ROW),
  w90  AS (ORDER BY charge_date
     ROWS BETWEEN 89  PRECEDING AND 
CURRENT ROW),
  w360 AS (ORDER BY charge_date
     ROWS BETWEEN 359 PRECEDING AND 
CURRENT ROW)
ORDER BY charge_date;

===== PAGE 31 =====
用均線的 μ、σ 預測「超支機率」
MA30 統計量
μ = 平均
σ = 標準差
預測剩餘期間
d 天 ≈ d·μ
波動 ≈ σ√d
與剩餘預算 B
相比較
z 分數
超支機率
= 風險量化
z = (B − d·μ) / (σ√d)　→　P(超支) = 1 − Φ(z)
把「會不會超支」變成一個 0–100% 的機率，這就是風險量化
假設剩餘期間每日花費近似常態：預測總額 d·μ、標準差 σ√d，算出超出全年剩餘預算的機率

===== PAGE 32 =====
超支機率的 SQL：常態分配尾端
-- 取最新一天的 MA30 μ、σ，
-- 與全年剩餘預算 B
WITH latest AS (
  SELECT ma30 AS mu, sd30 AS sigma
  FROM ma_view
  ORDER BY charge_date DESC LIMIT 1
),
rem AS (
  SELECT (b.amount - x.spent) AS 
b_left,
    (b.period_end - CURRENT_DATE) AS 
days_left
  FROM budget b, spent_view x
)
SELECT
  l.mu, l.sigma, r.b_left, 
r.days_left,
  -- z = (B - days*mu) / 
(sigma*sqrt(days))
  (r.b_left - r.days_left * l.mu)
    / (l.sigma * sqrt(r.days_left)) 
AS z,
  -- P(超支) = 1 - Φ(z)
  1 - (0.5 * (1 + erf(
    ((r.b_left - r.days_left*l.mu)
     /(l.sigma*sqrt(r.days_left)))
    / sqrt(2)))) AS prob_overspend
FROM latest l, rem r;
